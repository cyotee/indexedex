# Review Notes

Store targeted code review findings here as Markdown files.

## Naming

Suggested:

- `YYYY-MM-DD_task-<N>_<short-title>.md`

Examples:

- `2026-01-10_task-13_uniswap-v2-deployVault.md`
- `2026-01-11_task-7_slipstream-stack-too-deep-refactor.md`
- `2026-01-12_area-fee-oracles_core-fee-paths.md`

## Template

Use the “Findings Template” section in [CODE_REVIEW_PLAN.md](../../CODE_REVIEW_PLAN.md).

For defining review plans for non-task areas, start from:

- `plans/AREA_PLAN_TEMPLATE.md`
