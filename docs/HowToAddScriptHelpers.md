# How to add Script_* deployment helper functions

Purpose
- Concrete, copy-pasteable instructions for adding Script_* helper functions used by `Script_Crane`-style scripts to deploy and register new components (facets, DFPkgs, packages, etc.). Follow these steps exactly to preserve deterministic CREATE3 deployment patterns and test compatibility.

Key principles
- ALWAYS deploy via the CREATE3 factory helpers (do not use `new`). Use `factory().create3(initCode, initData, salt)` and the convention salt `keccak256(abi.encode(type(YourContract).name))`.
- Register instances with `registerInstance(chainid, INITCODE_HASH, address)` and call `declare(builderKey_Crane(), "name", address)` for VM labels.
- Cache/look up instances by chain id using `chainInstance(chainid, INITCODE_HASH)` via the existing chain-instance helpers.
- Keep public API consistent: provide (1) chain-aware setter, (2) chain-aware getter, (3) simple setter, (4) lazy deployer/getter.
- Add an INITCODE and INITCODE_HASH constant to `contracts/crane/constants/CraneINITCODE.sol` for any new deployable target.

Function templates (recommended pattern)

1) Chain-aware setter (register + declare)

```solidity
function fooFacet(uint256 chainid, FooFacet foo_) public virtual returns (bool) {
    registerInstance(chainid, FOO_FACET_INIT_CODE_HASH, address(foo_));
    declare(builderKey_Crane(), "fooFacet", address(foo_));
    return true;
}
```

2) Simple setter (current chain wrapper)

```solidity
function fooFacet(FooFacet foo_) public virtual returns (bool) {
    fooFacet(block.chainid, foo_);
    return true;
}
```

3) Chain-aware getter (returns stored instance)

```solidity
function fooFacet(uint256 chainid) public virtual view returns (FooFacet foo_) {
    foo_ = FooFacet(chainInstance(chainid, FOO_FACET_INIT_CODE_HASH));
    return foo_;
}
```

4) Lazy deployer / parameterless getter

```solidity
function fooFacet() public virtual returns (FooFacet foo_) {
    if (address(fooFacet(block.chainid)) == address(0)) {
        // encode init args if any
        bytes memory initData = abi.encode(/* args */);
        // deploy via factory().create3
        foo_ = FooFacet(
            factory().create3(
                FOO_FACET_INITCODE,
                initData,
                keccak256(abi.encode(type(FooFacet).name))
            )
        );
        // register and declare
        fooFacet(block.chainid, foo_);
    }
    return fooFacet(block.chainid);
}
```

Notes on init data and salt
- Init data must match the contract/package constructor or diamond factory `deploy` args. Use `abi.encode(...)` with the expected struct/layout.
- Use `keccak256(abi.encode(type(Contract).name))` as salt to remain consistent with other Script_* helpers.

Constants and registration
- Add the new INITCODE and INITCODE_HASH constants in `contracts/crane/constants/CraneINITCODE.sol`.
- Use the same `_INITCODE` and `_INITCODE_HASH` naming convention used across Crane.

Testing & scripts
- Add IFacet tests where appropriate. Facet tests must implement `facetTestInstance()`, `controlFacetInterfaces()`, and `controlFacetFuncs()`.
- Scripts that use these helpers should inherit `Script_Crane` (or the appropriate `ScriptBase_Crane_*`) and implement `run()` that invokes specific helper methods.


Checklist before merging or creating PR
- [ ] Added INITCODE and INITCODE_HASH to `contracts/crane/constants/CraneINITCODE.sol`.
- [ ] Implemented the 4 helper functions in the appropriate ScriptBase or Script_Crane mixin.
- [ ] Used `factory().create3(...)` for any deployment code.
- [ ] Registered instance with `registerInstance(...)` and called `declare(...)`.
- [ ] Added or updated tests inheriting the correct TestBase.
- [ ] Updated `remappings.txt` or `foundry.toml` if you added new in-repo libs.

Common pitfalls
- Forgetting to register/declare the instance (breaks downstream lookups).
- Using `new` which produces non-deterministic addresses and fails the test harness.
- Mismatching initData encoding vs package/constructor signature.
- Not adding the INITCODE_HASH constant (used by chainInstance lookups).
