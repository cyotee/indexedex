# Configuring Wagmi CLI to Generate Hooks for Multiple ERC20 and ERC4626 Contracts

This guide explains how to configure the Wagmi CLI to generate React Hooks for multiple instances of ERC20 tokens and ERC4626 vaults, using JSON files for ABIs (from Foundry artifacts) and deployment addresses. It assumes you have:
- A React project with Wagmi CLI installed (`pnpm add -D @wagmi/cli`).
- A Foundry project where `forge build` has generated ABI artifacts in `out/`.
- A JSON file (`addresses.json`) containing deployment addresses, generated by your Foundry deployment script.
- Familiarity with Wagmi's hook generation process.

The goal is to configure `wagmi.config.ts` to:
1. Import ABIs from Foundry-generated JSON files.
2. Import contract addresses from `addresses.json`.
3. Define multiple contract instances (three ERC20 tokens and three ERC4626 vaults) sharing their respective ABIs, generating unique hooks for each.

## Prerequisites
- **ABI Files**: Foundry's `out/ERC20Token.sol/ERC20Token.json` and `out/ERC4626Vault.sol/ERC4626Vault.json` copied to your React project's `abis/` folder (e.g., `abis/ERC20Token.json`, `abis/ERC4626Vault.json`). Extract the ABI with:
  ```bash
  jq .abi out/ERC20Token.sol/ERC20Token.json > ../react-project/abis/ERC20Token.json
  jq .abi out/ERC4626Vault.sol/ERC4626Vault.json > ../react-project/abis/ERC4626Vault.json
  ```
  (Adjust paths as needed; install `jq` if necessary.)
- **Addresses File**: A JSON file (`addresses.json`) in your React project root, with a structure like:
  ```json
  {
    "TokenA": "0x1234...abcd",
    "TokenB": "0x5678...efgh",
    "TokenC": "0x9abc...ijkl",
    "VaultA": "0xdef0...1234",
    "VaultB": "0x5678...abcd",
    "VaultC": "0x9abc...def0"
  }
  ```
- **Chain**: Sepolia (or adjust for your chain).
- **Output**: Hooks generated in `src/generated.ts`.

## Step 1: Create `wagmi.config.ts`
In your React project root, create or edit `wagmi.config.ts`. This file will:
- Import ABIs from `abis/ERC20Token.json` and `abis/ERC4626Vault.json`.
- Import addresses from `addresses.json`.
- Define contract instances for three ERC20 tokens (`TokenA`, `TokenB`, `TokenC`) and three ERC4626 vaults (`VaultA`, `VaultB`, `VaultC`), each with unique names and addresses but sharing their respective ABIs.
- Use the `react` plugin to generate hooks.

Here’s the complete configuration:

```typescript
import { defineConfig } from '@wagmi/cli'
import { react } from '@wagmi/cli/plugins'
import { sepolia } from 'wagmi/chains'

// Import ABIs from Foundry-generated JSON artifacts
const erc20Abi = require('./abis/ERC20Token.json').abi
const erc4626Abi = require('./abis/ERC4626Vault.json').abi

// Import deployment addresses
const addresses = require('./addresses.json')

export default defineConfig({
  out: 'src/generated.ts', // Output file for hooks
  contracts: [
    // ERC20 Tokens (share the same ABI)
    {
      name: 'TokenA',
      abi: erc20Abi,
      address: {
        [sepolia.id]: addresses.TokenA,
      },
    },
    {
      name: 'TokenB',
      abi: erc20Abi,
      address: {
        [sepolia.id]: addresses.TokenB,
      },
    },
    {
      name: 'TokenC',
      abi: erc20Abi,
      address: {
        [sepolia.id]: addresses.TokenC,
      },
    },
    // ERC4626 Vaults (share the same ABI)
    {
      name: 'VaultA',
      abi: erc4626Abi,
      address: {
        [sepolia.id]: addresses.VaultA,
      },
    },
    {
      name: 'VaultB',
      abi: erc4626Abi,
      address: {
        [sepolia.id]: addresses.VaultB,
      },
    },
    {
      name: 'VaultC',
      abi: erc4626Abi,
      address: {
        [sepolia.id]: addresses.VaultC,
      },
    },
  ],
  plugins: [
    react(), // Generates hooks like useReadTokenABalance, useWriteVaultADeposit
  ],
})
```

## Step 2: Explanation of Configuration
- **ABI Imports**:
  - `require('./abis/ERC20Token.json').abi` extracts the ABI array from the Foundry artifact. The `.abi` key is necessary because Foundry’s JSON includes other data (e.g., bytecode).
  - Similarly for `erc4626Abi`. Ensure the JSON files are in `abis/` or adjust the path.
- **Address Imports**:
  - `require('./addresses.json')` loads your deployment addresses. The keys (`TokenA`, `TokenB`, etc.) match the contract names in the `contracts` array.
- **Contracts Array**:
  - Each entry defines a contract instance with:
    - `name`: A unique identifier (e.g., `TokenA`) used to prefix hooks (e.g., `useReadTokenABalance`).
    - `abi`: The shared ABI (`erc20Abi` for tokens, `erc4626Abi` for vaults).
    - `address`: A chain-to-address mapping (e.g., `[sepolia.id]: addresses.TokenA`). Use `sepolia.id` from `wagmi/chains` for type safety.
  - All tokens share `erc20Abi`; all vaults share `erc4626Abi`. This ensures hooks use the correct function signatures but target different addresses.
- **Plugins**:
  - `react()` generates hooks for each contract’s ABI functions (e.g., `balanceOf` for ERC20, `deposit` for ERC4626).
- **Output**:
  - `out: 'src/generated.ts'` specifies where hooks are written.

## Step 3: Generate Hooks
Run:
```bash
pnpm wagmi generate
```
This creates `src/generated.ts` with hooks like:
- ERC20: `useReadTokenABalance`, `useWriteTokenATransfer`, `useReadTokenBBalance`, etc.
- ERC4626: `useReadVaultATotalAssets`, `useWriteVaultADeposit`, `useReadVaultBMaxRedeem`, etc.

## Step 4: Automate Generation
Add to `package.json`:
```json
{
  "scripts": {
    "predev": "wagmi generate",
    "dev": "next dev" // Or your dev command
  }
}
```
Run `pnpm dev` to regenerate hooks on start.

## Step 5: Using Hooks in Your App
Ensure your React app is configured with Wagmi (e.g., wrapped in `<WagmiProvider>` with Sepolia support). Example component:
```tsx
'use client'

import { useReadTokenABalance, useReadVaultATotalAssets } from './generated'
import { useAccount } from 'wagmi'

function App() {
  const { address } = useAccount()
  const { data: tokenABalance } = useReadTokenABalance({ args: [address!] })
  const { data: vaultAAssets } = useReadVaultATotalAssets()

  return (
    <div>
      <p>Token A Balance: {tokenABalance?.toString()}</p>
      <p>Vault A Total Assets: {vaultAAssets?.toString()}</p>
    </div>
  )
}
```

## Troubleshooting
- **Missing JSON Files**: Ensure `abis/ERC20Token.json` and `abis/ERC4626Vault.json` exist and contain valid ABI arrays. Check with `cat abis/ERC20Token.json`.
- **Address Errors**: Verify `addresses.json` keys match `wagmi.config.ts` names (`TokenA`, `VaultA`, etc.) and contain valid addresses.
- **Chain ID**: Ensure `sepolia.id` (11155111) matches your deployment chain. Import other chains from `wagmi/chains` if needed.
- **Hook Naming Conflicts**: Unique `name` fields prevent conflicts (e.g., `TokenA` vs. `TokenB`).
- **Verbose Debugging**: Run `pnpm wagmi generate --verbose` to diagnose issues.

## Notes
- **Scalability**: Add more instances by extending the `contracts` array with new names and addresses.
- **Multi-Chain**: Add addresses for other chains (e.g., `[mainnet.id]: addresses.TokenA_mainnet`).
- **ABI Updates**: If you modify your contracts, re-run `forge build`, update `abis/`, and regenerate hooks.

This configuration ensures Wagmi generates hooks for each contract instance, correctly associating shared ABIs with unique addresses.
